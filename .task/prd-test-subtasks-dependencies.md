# 测试用 PRD：多子任务与依赖关系

## 1. 文档信息

- 产品名称：Wise 任务协同模块（测试需求）
- 文档版本：v1.0-test
- 编写日期：2026-04-23
- 使用目的：验证“PRD -> 任务拆分”在子任务、依赖关系、并行组、关键路径场景下的正确性

---

## 2. 背景与目标

当前需要一份结构完整、依赖关系明确的测试 PRD，用于验证系统是否能：

1. 正确识别父任务与子任务层级；
2. 正确推导任务间前置依赖；
3. 给出可并行执行任务组；
4. 计算关键路径；
5. 产出可直接执行的开发与测试清单。

---

## 3. 业务场景

用户希望在应用中新增“任务看板 + 评论 + 通知”能力：

- 项目成员可在看板中创建/移动任务卡片；
- 任务可添加评论，支持 @ 提及；
- 当任务状态变化或被提及时，相关成员收到通知。

---

## 4. 功能需求（含子任务与依赖）

### Epic A：任务看板基础能力（M）

- 目标：支持任务卡片的创建、编辑、状态流转
- 前置依赖：无
- 子任务：
  - A1（S）数据模型设计：定义 Board/Column/Card 结构
    - 前置依赖：无
  - A2（S）后端接口：新增看板查询与卡片更新 API
    - 前置依赖：A1
  - A3（S）前端看板页面：展示列与卡片列表
    - 前置依赖：A2
  - A4（S）拖拽状态变更：卡片跨列移动并保存
    - 前置依赖：A3
- 验收标准（DoD）：
  - 可创建至少 3 个状态列（待办/进行中/已完成）
  - 卡片可拖拽跨列并持久化
  - 刷新页面后状态一致

### Epic B：任务评论能力（M）

- 目标：支持任务评论与评论列表
- 前置依赖：Epic A 完成
- 子任务：
  - B1（S）评论表结构与存储设计
    - 前置依赖：A1
  - B2（S）评论 API（增/查）
    - 前置依赖：B1
  - B3（S）评论输入与列表组件
    - 前置依赖：A3, B2
  - B4（S）评论分页与时间排序
    - 前置依赖：B3
- 验收标准（DoD）：
  - 同一任务可连续发布评论
  - 评论列表按时间倒序
  - 新评论 1 秒内可见

### Epic C：通知中心（L，需拆分）

- 目标：任务变更与 @ 提及时发送通知
- 前置依赖：Epic A、Epic B
- 拆分任务：
  - C1（M）事件定义与触发机制
    - 子任务：
      - C1-1（S）定义事件类型（状态变更、评论新增、@提及）
      - C1-2（S）在任务与评论流程中埋点触发事件
    - 前置依赖：A4, B3
  - C2（M）通知分发服务
    - 子任务：
      - C2-1（S）通知接收人计算规则
      - C2-2（S）通知消息入库与未读状态
    - 前置依赖：C1
  - C3（M）通知前端展示
    - 子任务：
      - C3-1（S）右上角通知角标与下拉面板
      - C3-2（S）通知已读/全部已读交互
    - 前置依赖：C2
- 验收标准（DoD）：
  - 任务状态变化触发通知
  - 评论中 @ 指定用户可收到通知
  - 通知可标记已读且数量同步更新

### Epic D：权限与审计（M）

- 目标：限制非成员操作并记录关键行为日志
- 前置依赖：Epic A
- 子任务：
  - D1（S）项目成员权限校验中间层
    - 前置依赖：A2
  - D2（S）前端按钮级权限控制（禁用/隐藏）
    - 前置依赖：D1, A3
  - D3（S）审计日志记录（卡片移动、评论删除、状态修改）
    - 前置依赖：D1, A4, B2
- 验收标准（DoD）：
  - 非成员无法修改任务状态
  - 关键动作均可查询审计日志

---

## 5. 全局依赖关系（用于 DAG 测试）

- A1 -> A2 -> A3 -> A4
- A1 -> B1 -> B2 -> B3 -> B4
- A4 + B3 -> C1 -> C2 -> C3
- A2 -> D1 -> D2
- D1 + A4 + B2 -> D3

---

## 6. 可并行任务组（用于并行规划测试）

- 并行组 G1：A1
- 并行组 G2：A2、B1（B1 仅依赖 A1）
- 并行组 G3：A3、B2、D1
- 并行组 G4：A4、B3、D2
- 并行组 G5：B4、C1、D3
- 并行组 G6：C2
- 并行组 G7：C3

---

## 7. 关键路径（预期）

用于测试算法输出是否正确，预期关键路径之一为：

`A1 -> A2 -> A3 -> A4 -> C1 -> C2 -> C3`

说明：若系统计算结果与该路径不同，需要检查依赖图构建逻辑或任务工时权重设置。

---

## 8. 非功能需求

- 性能：看板首页加载时间 < 2 秒（100 条卡片以内）
- 稳定性：通知分发失败率 < 0.5%
- 安全：接口需基于用户身份校验项目访问权限
- 可观测性：核心链路埋点覆盖率 100%（创建任务、评论、通知发送）

---

## 9. 测试验收清单（勾选）

### 功能验收

- [ ] 可创建卡片并在看板展示
- [ ] 卡片可拖拽跨列且刷新后不丢失
- [ ] 卡片评论可新增、可查看、按时间排序
- [ ] 评论 @ 用户后，对方收到通知
- [ ] 通知可单条已读和全部已读
- [ ] 非成员无法进行受限操作

### 依赖与拆分验收

- [ ] 能识别 Epic/任务/子任务三级结构
- [ ] 能识别多前置依赖（如 D3 依赖 D1+A4+B2）
- [ ] 能计算并行任务组
- [ ] 能计算关键路径并输出可读结果
- [ ] L 级任务（Epic C）被继续拆分为 M/S 级任务

### 导出验收

- [ ] 可导出为 Markdown 清单
- [ ] 导出内容包含任务大小、依赖、DoD
- [ ] 导出结果可直接用于开发排期与测试执行
