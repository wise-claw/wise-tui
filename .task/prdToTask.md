# PRD -> Task 可执行任务清单

## 目标

支持 PRD 来源从原有文本输入扩展为以下 3 种：

- 手动输入文本
- 复制粘贴 Markdown 内容
- 复制链接后通过 Claude Code 拉取内容

并统一生成可执行任务拆分（任务大小、子任务、依赖顺序），用于生码完成后的逐项确认。

## 任务拆分规则（执行前统一约定）

- 任务大小
  - `S`: 0.5-1 人天，可独立开发+自测
  - `M`: 1-3 人天，需拆分为 2-4 个子任务
  - `L`: 3-5 人天，不可直接开发，必须继续拆分
- 每个任务必须包含
  - 任务描述
  - 前置依赖
  - 验收标准（DoD）
  - 产出物（代码/文档/配置）

## 左侧需求来源 -> 右侧详细任务（可执行）

### 1. 输入来源识别层（M）

- 需求来源：支持粘贴 Markdown 或 URL
- 前置依赖：无
- 子任务：
  - [x] `S` 识别输入类型：`plain_text` / `markdown` / `url`
  - [x] `S` 增加输入校验（空输入、超长输入、非法 URL）
  - [x] `S` 为输入类型输出统一元数据（sourceType、rawText、rawUrl）
- DoD：
  - 可稳定识别三种输入来源
  - 非法输入有明确错误提示
  - 输出结构可被下游解析模块直接消费

### 2. Markdown 解析层（M）

- 需求来源：用户复制 PRD 的 MD 内容直接使用
- 前置依赖：任务 1
- 子任务：
  - [x] `S` 提取一级结构：目标/场景/功能/非功能/验收
  - [x] `S` 支持常见 Markdown 元素（标题、列表、表格、代码块）
  - [x] `S` 增加结构缺失容错（字段缺失时给默认占位）
- DoD：
  - 至少支持常见 PRD Markdown 格式
  - 结构化结果字段完整（缺失字段有 fallback）
  - 错误输入不会导致流程中断

### 3. URL 拉取层（Claude Code）（M）

- 需求来源：用户直接粘贴链接自动拉取 PRD
- 前置依赖：任务 1
- 子任务：
  - [x] `S` 链接可访问性检查（协议、域名、超时）
  - [x] `S` 通过 Claude Code 拉取正文内容
  - [x] `S` 对拉取内容做清洗（导航噪音、空白、重复段落）
  - [x] `S` 拉取失败兜底：引导用户粘贴正文
- DoD：
  - 对可访问链接可完成正文拉取并进入解析流程
  - 拉取失败有可操作的回退路径
  - 拉取结果可追踪（记录标题、来源、时间）

### 4. PRD 标准化层（M）

- 需求来源：不同来源统一为同一套 PRD Schema
- 前置依赖：任务 2、任务 3
- 子任务：
  - [x] `S` 定义统一 Schema：背景/目标/场景/功能/非功能/验收
  - [x] `S` 补充字段映射规则（MD 字段别名、URL 抓取字段）
  - [x] `S` 增加标准化校验（必填、长度、去重）
- DoD：
  - 任一来源可输出统一 PRD 对象
  - 校验失败可定位具体字段问题
  - 下游任务拆分不感知来源差异

### 5. 任务拆分引擎（L -> 拆分为 3 个 M）

- 需求来源：按 PRD 自动拆分可执行任务
- 前置依赖：任务 4
- 子任务 A（M）任务粒度控制：
  - [x] `S` 实现 S/M/L 判定逻辑
  - [x] `S` 对 L 任务自动二次拆分
  - [x] `S` 为每个任务生成建议工时
- 子任务 B（M）任务依赖生成：
  - [x] `S` 提取任务前置条件
  - [x] `S` 构建 DAG 依赖关系
  - [x] `S` 产出关键路径与可并行任务组
- 子任务 C（M）任务明细生成：
  - [x] `S` 生成每个任务的子任务列表
  - [x] `S` 生成验收标准（DoD）
  - [x] `S` 生成输出格式（左来源右任务）
- DoD：
  - 任一 PRD 都可输出结构化任务清单
  - 每个任务均含大小、依赖、子任务、DoD
  - 输出能直接进入开发执行与验收

### 6. 人机协同优化（Claude Code 对话调整）（M）

- 需求来源：支持对拆分结果继续对话优化
- 前置依赖：任务 5
- 子任务：
  - [x] `S` 提供“重算拆分”入口（基于最新对话上下文）
  - [x] `S` 支持定向调整（只改任务大小/只改依赖/只改 DoD）
  - [x] `S` 保留版本快照（v1/v2/v3）便于回滚对比
- DoD：
  - 用户可通过对话精准调整拆分结果
  - 每次调整有版本记录和差异可追溯
  - 最终版本可一键导出

### 7. 输出与验收清单（S）

- 需求来源：生码后可逐项确认是否完成
- 前置依赖：任务 6
- 子任务：
  - [x] `S` 输出标准 Markdown 清单（勾选式）
  - [x] `S` 增加“开发完成确认”区（自测/联调/回归）
  - [x] `S` 增加“上线前确认”区（文档/风险/回滚）
- DoD：
  - 文档可直接作为交付验收单
  - 开发与产品可共用同一确认视图

```text
请基于输入的 PRD（可能是 Markdown 或 URL 拉取内容）生成任务拆分，要求：
1) 每个任务标注 S/M/L；L 任务必须继续拆分；
2) 每个任务必须包含：子任务、前置依赖、DoD；
3) 输出“左侧需求来源 -> 右侧详细任务”结构；
4) 给出关键路径和可并行任务组；
5) 输出为可勾选的 Markdown 验收清单。
```
