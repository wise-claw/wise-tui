# PRD 任务拆分 V2 使用操作清单

更新时间：2026-04-24

## 1) 面板入口

- 打开应用中的 `任务拆分` 面板（`PrdTaskSplitPanel`）。
- 绑定目标仓库（建议先绑定，便于写入 `.wise/prd-runs` 快照）。

## 2) 首轮拆分

- 在左侧输入 PRD（文本/Markdown/URL）。
- 点击 `拆分`。
- 预期：
  - 生成任务列表；
  - 右侧出现校验信息；
  - 运行会自动做策略路由（`policyId`）。

## 3) V2 深化（核心）

- 点击 `更多操作` -> `优化拆分结果`。
- 勾选反馈标签（如粒度/依赖/覆盖）。
- 选择重算模式：
  - `全量重算`：整体优化，改动范围大；
  - `增量重算`：仅改受影响范围，改动更稳。
- 点击 `用 Claude 深化`。

## 4) 如何判断是否可用

查看右侧 `策略与门禁状态`：

- `policy`：当前策略 ID（如 `feature_domain_first`）。
- `mergeDecision`：
  - `block`：必须先修复硬错误；
  - `warn`：可继续，但有风险告警；
  - `allow`：可进入导出/交付。
- `hardErrors`：硬错误数量（目标 0）。
- `softWarnings`：软告警数量（越少越好）。

## 5) 增量模式说明

- 选择 `增量重算` 后，系统会计算 `affectedRequirements / affectedTaskIds / boundaryTaskIds`。
- 合并阶段会做边界保护：
  - 非受影响任务的映射改写会被忽略并告警；
  - 触及边界任务会告警（便于人工确认）。

## 6) 学习与回放

- 深化后会沉淀学习样本到 `.wise/prd-split-learning-samples.jsonl`。
- 后续深化会按 `prdFingerprint + feedbackTags` 自动召回 Top-K 案例注入提示词。

## 7) 影子评测（离线对比）

在仓库根目录执行：

```bash
bun extras/check-prd-split-evolution.mjs --policies=feature_domain_first,user_journey_first,tech_layer_first --topk=20 --out=.wise/prd-split-policy-compare.json
```

输出包括：

- 同 PRD 多策略横向对比；
- 策略总榜（rubric 分、hardErrors、softWarnings、taskCount）。

## 8) 推荐日常流程（最短）

1. 拆分（默认策略）  
2. 反馈标签 + 增量深化  
3. 看 `mergeDecision`（至少到 `warn`，最好 `allow`）  
4. 导出结果  
5. 每周跑一次影子评测脚本对比策略

## 9) 常见问题

- 看不到快照文件：先确认仓库已绑定；未绑定时不会写 `.wise/prd-runs`。
- 一直 `block`：优先修依赖环、缺失映射、非法引用等硬错误。
- 提示词太长：控制反馈说明长度，优先保留最关键 1-2 个问题标签。
