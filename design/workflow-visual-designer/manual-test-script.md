# Workflow 可视化设计器手工验证脚本

## 1. 准备

- 启动应用：`bun run desktop:dev`（或前端模式 `bun run dev`）
- 确认已有至少 1 个可用员工（用于审批阶段配置）
- 打开任一 Claude 会话页面，进入“工作流配置”

---

## 2. Phase 1 验证（设计器骨架）

1) 模式切换
- 在工作流配置弹窗切换“表单模式 / 设计器模式”
- 预期：切换正常，无报错；表单模式功能保持可用

2) 图编辑
- 在设计器中拖拽节点、连线
- 预期：节点位置与连线变化实时可见

3) 保存草稿
- 点击“创建草稿/保存草稿”
- 预期：模板保存成功；重进编辑可回显图或阶段转换图

---

## 3. Phase 2 验证（发布与校验）

1) 发布前校验
- 构造非法图（例如删除 start 节点）
- 点击“发布模板”
- 预期：出现结构化错误列表；错误可点击定位节点/边

2) 错误分组与建议
- 查看校验面板
- 预期：按“流程结构/节点/连线”分组，且每条有修复建议

3) 状态可见化
- 发布合法图后返回列表
- 预期：模板“图状态”显示 `published`；发布成功提示可见

4) 草稿与发布区分
- 在设计器修改图后点“保存草稿”
- 预期：图状态保持或变为 `draft`（取决于当前实现策略），不应误报已发布

---

## 4. Phase 3 验证（执行态最小闭环）

1) 任务运行图显示
- 创建/选择一个 workflow 任务，打开任务时间线区域
- 预期：每个任务显示只读流程图

2) 当前阶段高亮
- 观察任务 `currentStageIndex` 对应节点
- 预期：该阶段节点高亮，路径边可见突出

3) 时间线 -> 图
- 点击最近事件（带阶段索引）
- 预期：流程图定位到对应阶段节点

4) 图 -> 时间线
- 点击流程图审批节点
- 预期：时间线事件按对应阶段过滤
- 点击“清除过滤”
- 预期：恢复全量事件

5) 事件展开
- 当事件超过 3 条时点击“展开N条/收起”
- 预期：事件列表按按钮状态切换

---

## 5. 回归检查

- 员工配置、任务创建、审批通过/退回功能可正常使用
- 旧表单工作流创建和编辑不受影响
- 页面无明显样式错位（亮/暗主题）

---
