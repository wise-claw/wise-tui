# PRD: 中栏多屏模式（2/4/6/8屏）

## 背景

当前中栏（chat 视图）仅支持"单屏"和"双屏"两种模式，双屏通过布尔值 `dualPaneEnabled` 控制，且只维护一个 secondary session。用户需要同时并行多个仓库/会话的能力，要求扩展为支持 2、4、6、8 屏。

## 需求

### 功能需求

1. **屏数切换**：中栏支持 1（关闭）、2、4、6、8 屏五种模式。
2. **网格布局**：多屏采用网格排列——2屏=1×2，4屏=2×2，6屏=2×3，8屏=2×4（行×列）。
3. **Topbar 下拉菜单**：原双栏切换按钮改为下拉菜单，可选 1/2/4/6/8 屏。
4. **快捷键循环**：Alt+K 在 1→2→4→6→8→1 之间循环切换。
5. **新增窗格初始化**：新增的窗格显示空屏状态（EmptyState），用户通过 Repo 选择器选择仓库后自动创建会话。
6. **会话保持**：切换屏数时，已有窗格的会话保持不变；减少屏数时，被移除窗格的会话从活跃列表中移除但不删除。
7. **窗口自适应**：窗口大小根据屏数自动调整，保证每个窗格有最小宽度（460px）。
8. **通知隔离**：每个窗格的通知（todo/追问/权限）独立隔离，与现有 `useDockSlice` 机制兼容。

### 非功能需求

- 不改变 `ViewMode` 状态机结构，多屏模式仅作用于 `chat` 视图。
- 保持 `invoke` 隔离规则：组件通过 service 层调用 Tauri IPC。
- CSS 使用 flexbox/grid，不引入新 UI 框架。
- 保持 Ant Design 为 UI 系统。

## 验收标准

- [ ] Topbar 出现多屏下拉菜单，可选 1/2/4/6/8 屏。
- [ ] 选择 2 屏时，中栏显示 1×2 网格（与当前双栏行为一致）。
- [ ] 选择 4 屏时，中栏显示 2×2 网格，新增 2 个空屏窗格。
- [ ] 选择 6 屏时，中栏显示 2×3 网格。
- [ ] 选择 8 屏时，中栏显示 2×4 网格。
- [ ] Alt+K 快捷键在 1→2→4→6→8→1 之间循环切换。
- [ ] 增加屏数时，新窗格显示空屏+Repo选择器。
- [ ] 减少屏数时，已有窗格会话不丢失。
- [ ] 窗口宽度随屏数自适应调整。
- [ ] 各窗格通知独立隔离。
